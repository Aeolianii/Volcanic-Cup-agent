# AI Story Foundry 功能实现文档

## 1. 功能总览

当前项目已实现 AI 互动叙事 MVP 主链路：

```text
故事输入 -> Story Bible 生成 -> 可玩性分析 -> 校验 -> 创建房间
-> 玩家加入 -> 选择角色 -> 开始故事 -> 玩家行动
-> Rule Engine 结算 -> World State 更新 -> NPC 行动
-> 事件触发 -> 章节推进 -> GM 叙事 -> 结局判断
```

## 2. 首页

入口文件：`src/app/page.tsx`

已实现能力：

- 创建故事，跳转 `/generate`。
- 输入房间号和玩家名加入房间。
- 快速进入 Demo 故事 `/generate?demo=true`。
- 在浏览器 `localStorage` 保存 `player_id` 和 `player_name`。

## 3. 故事生成

入口文件：

- 页面：`src/app/generate/page.tsx`
- API：`src/app/api/stories/generate/route.ts`

已实现能力：

- 支持故事创意输入。
- 支持完整剧本文本导入。
- 支持题材、开场、结局方向、角色、世界观等结构化字段。
- 将输入归一化为 `StorySeed`。
- 执行可玩性分析并生成 `PlayabilityReport`。
- 调用 Story Bible Generator 生成 `StoryBible`。
- 将可玩性结果应用到 Story Bible。
- 校验 Story Bible。
- 校验通过后写入 RoomManager，供创建房间使用。
- 支持 Demo Story Bible 快速加载。

主要输出：

- `story_bible`
- `validation`
- `playability`
- `success`
- `error`

## 10. 行动系统

相关文件：

- `src/types/action.ts`
- `src/engine/actionParser.ts`
- `src/app/api/rooms/[roomId]/actions/route.ts`
- `src/lib/llmProvider.ts`

已实现输入来源：

- `suggested_action`
- `free_action`
- `chat`
- `npc_planner`

已实现行动类别：

- 社交：`talk`、`persuade`、`threaten`、`deceive`、`ally`、`betray`、`confess`
- 调查：`investigate`、`search`、`track`、`eavesdrop`、`interrogate`、`decode`、`spy`、`divination`、`gather_intelligence`
- 政治：`command`、`summon_meeting`、`gain_support`、`coup`、`impeach`、`appoint`
- 战斗：`attack`、`assassinate`、`duel`、`ambush`、`defend`、`execute`、`sacrifice`
- 资源：`buy`、`trade`、`steal`、`transport`、`build`

行动处理流程：

```text
输入文本/推荐按钮
  -> 解析为 StructuredAction
  -> 规范化目标
  -> 回合权限检查
  -> Rule Engine
  -> World State 更新
  -> NPC 回合与 GM 平衡
  -> 事件、章节、结局检查
  -> GM 叙事与新推荐行动
```

## 11. Rule Engine

相关文件：

- `src/engine/ruleEngine.ts`
- `src/engine/worldStateEngine.ts`
- `src/engine/deathPolicy.ts`
- `src/engine/turnSystem.ts`

已实现能力：

- 权限检查。
- 风险等级与成功率结算。
- 指标变更。
- flag 设置与清除。
- 关系变化。
- 线索、证据、已知事实写入。
- 事件触发标记。
- 位置变化。
- Active Modifier 应用与衰减。
- NPC Runtime State 更新。
- 角色状态变化：`alive`、`dead`、`missing`、`imprisoned`、`defeated`、`setback`。
- 幽灵模式限制：死亡后只可旁观，不能行动或改变世界状态。
- 非死亡题材下将高风险后果转为题材安全的挫败。

## 12. NPC Agent 系统

相关文件：

- `src/engine/npcKnowledgeFilter.ts`
- `src/engine/npcPlanner.ts`
- `src/engine/npcTurnSystem.ts`
- `src/types/action.ts`
- `src/lib/llmProvider.ts`

已实现能力：

- 每个 NPC 拥有持久 Runtime State。
- NPC 按行动频率参与回合。
- NPC 只能读取局部视野，不读取完整 Story Bible。
- NPC 可基于目标、记忆、已知事实、关系、威胁目标和玩家公开行动生成行动提案。
- NPC 行动必须交由 Rule Engine 审核。
- 支持阻碍调查、误导玩家、隐藏证据、嫁祸、影响指标、影响 NPC、保护秘密、推进计划。
- 限制连续针对同一玩家，避免体验过度挫败。
- 允许制造假线索，但不能永久删除关键线索。

## 13. AI GM 系统

相关文件：

- `src/engine/aiGM.ts`
- `src/lib/llmProvider.ts`
- `src/mock/mockAIProvider.ts`

已实现能力：

- 生成开场叙事。
- 解释行动结果。
- 汇总 NPC 行动影响。
- 结合指标变化生成连续反馈。
- 根据事件触发生成事件叙事。
- 生成推荐行动。
- 生成结局叙事。

约束：

- AI GM 不直接修改 World State。
- AI GM 输出叙事、建议事件、揭示信息、推荐行动等文本/建议内容。
- 真实状态变化仍由 Rule Engine 和 World State Engine 完成。

## 14. 事件、章节与结局

相关文件：

- `src/engine/eventTriggerSystem.ts`
- `src/engine/storyController.ts`
- `src/engine/endingJudge.ts`
- `src/app/api/rooms/[roomId]/endings/check/route.ts`
- `src/app/ending/[roomId]/page.tsx`

已实现能力：

- 根据 World State 中的 flag、metric、事件状态触发事件。
- 事件触发后应用效果：设置 flag、修改指标、添加知识、揭示事件、切换位置。
- 根据章节进入条件推进章节。
- 根据结局条件和优先级判断结局。
- 拆分阵营胜利和个人胜利的运行时模型。
- 到达结局后跳转结局页，显示结局叙事和状态。

## 17. 已知限制

- README 和部分源码中文显示存在历史编码异常，需要后续统一 UTF-8 清理。
- 当前房间、状态、Story Bible 为内存存储，不具备生产持久化能力。
- Socket.IO 依赖已预留，但实时同步尚未完成。
- Story Bible 生成当前仍有 mock/provider 混合路径，生产环境需要更严格的失败策略。
- 测试主要集中在构建和冒烟，Rule Engine 与 NPC 行为仍需要更细粒度单元测试。
