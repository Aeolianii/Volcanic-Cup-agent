# 鏇存柊鏃ュ織

## v1.5 - 2026-05-24

### NPC Agent 对抗行动系统

- 新增独立 NPC Runtime State：每个 NPC 持有核心目标、阶段目标、当前计划、长期记忆、已知事实、疑似事实、关系快照、威胁目标、保护秘密、行动频率和自身 active modifiers。
- 新增 NPC 回合系统：玩家每回合行动，NPC 默认每 2 回合行动一次，按 `Observe -> Think -> Plan -> Act` 管线主动参与世界模拟。
- NPC Planner 只接收 Knowledge Filter 生成的局部视野，不能读取完整 Story Bible、结局条件、未来章节、其他 NPC 秘密或玩家私密行动。
- AI NPC 会根据玩家可观察行动、公开/条件可见指标、自己的目标、记忆和威胁评估生成行动提案，而不是作为聊天机器人等待玩家触发。
- 支持 NPC 对抗行为：阻碍调查、误导玩家、隐藏证据、嫁祸玩家、影响指标、影响 NPC、保护秘密和推进隐藏计划。

### Rule Engine 与公平性

- NPC 行动全部先变成行动提案，再由 Rule Engine 审核和结算；NPC 不能直接修改 World State。
- 玩家行动成功率公式加入 NPC 干扰项，并保留成功率下限，避免 NPC 行动造成纯粹挫败。
- 支持 NPC 伪造假证据：当可见真相/调查进度过高且威胁 NPC 目标时，NPC 可生成假线索并让真相进度小幅回落。
- 加入 `max_consecutive_target = 2` 规则，避免同一 NPC 无限锁定同一玩家。
- NPC 可延迟、隐藏或移动线索，但不会永久删除关键线索，设计目标是提升故事张力而非击败玩家。

### 导入故事泛化

- NPC Agent 系统不再依赖 Demo 的固定指标和地点，会从导入故事的 metric id、label、NPC 目标、秘密、记忆和玩家可观察行动中动态推断。
- 支持导入故事中的自定义调查进度、势力、压力、怀疑、稳定、信任等指标。
- 玩家调查目标若触及某个 NPC 自身目标、秘密目标、初始知识或记忆，该 NPC 可合理观察或推断到威胁。

### 玩家视图与剧透控制

- 事件追踪面板只显示玩家已解锁/已触发事件，不再提前展示后续故事节点。
- 已知事实和线索输出增加清洗逻辑，修复 `Action completed`、`Clue found`、`current_location`、`ancient_runes_discovered` 等内部 id 泄露。
- GM 叙事上下文接入 NPC 行动结果，可总结玩家行动、NPC 世界影响、指标变化、故事推进和新线索。

### 验证

- 已通过 `npx tsc --noEmit`。
- 已通过 `npm run build`。
- 已在本地 `http://localhost:3000` 验证事件节点隐藏、线索文本清洗和 Next 缓存重启恢复。
