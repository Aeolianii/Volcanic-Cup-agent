# 更新日志

## v1.3 - 2026-05-24

### 隐藏指标与模糊化反馈

- 新增隐藏/条件指标系统：后台仍然记录真实数值（如 suspicion +5），但玩家叙事中不再暴露具体数值和内部 key。
- 隐藏指标变化以"暗流变化"形式呈现模糊化反馈，例如"你的举动引起了一些人的怀疑"，而非"怀疑值上升5"。
- AI GM 上下文通过 `implicit_effects` 传递隐藏指标效果，并增加约束禁止输出隐藏指标名称、内部 key 和具体数值。
- GM 叙事中的"局势变化"仅显示玩家可见的公开指标，隐藏指标不再泄露到玩家文本。

### 推荐行动生成改进

- 导入故事的推荐行动必须来自 AI，结合 Story Bible + World State + 当前章节 + 已触发事件 + 上一行动结果生成，不再使用本地模板冒充。
- 导入故事 AI 未成功返回推荐时保持空/未生成，不再用本地模板补 3 条。
- 强化 LLM 推荐行动 prompt，要求必须基于当前剧情生成，不得重复上一行动，不得使用无关固定模板。
- 修复兜底叙事路径漏传 `lastAction` 导致推荐行动重复返回刚点击过的旧行动的问题。
- 推荐行动提交失败或网络失败时恢复原列表，避免临时移除后丢失。

### Rule Engine 与行动优势系统

- 新增行动优势（Action Advantage）机制：成功行动后积累动量、目标锁定和行动类别优势，为后续行动提供骰面加成。
- 调整风险等级阈值：低风险 20（+15）、中风险 40（+5）、高风险 60（-5），整体提升行动成功率。
- 调查类行动根据具体方法差异化推进真相进度（窃听/审讯 +20、搜索/追踪 +15、交叉验证 +15、默认 +12）。
- 社交类行动（交谈、说服、审讯、欺骗）成功后也推进真相进度，NPC 目标 +8、非 NPC +5。
- 自我梳理行动成功后不再重复添加已知事实。

### 事件效果与章节推进

- 事件触发后自动应用事件效果（EventEffect）到 World State，支持设置标记、修改指标、添加知识和切换位置。
- 行动结算后检查章节推进条件，满足时自动切换章节并在 World State 中记录章节启动标记。
- 事件触发系统支持 `flag_` 前缀的条件判断，可检测动态标记是否存在。

### AI Provider 与 GM 叙事

- 移除 Mock Provider 自动降级逻辑，GM 叙事和推荐行动不再因缺少 API Key 而静默走 mock。
- GM 叙事改用严格 JSON 解析（`chatJSONStrict`），不再提供 mock fallback，AI 失败时明确返回错误。
- AI GM 上下文中 World State 指标使用 Story Bible 中的 `label` 而非内部 `metric_id`，提升叙事可读性。
- 优化行动后叙事 prompt，增加对 `implicit_effects` 的处理指令和推荐行动质量约束。

### Demo：失落圣杯之夜

- 修复追问主教推荐行动的文案和上下文，改为"向主教核对大法师说法"并携带具体线索上下文，避免玩家误解为目标混淆 bug。
- 验证推荐行动刷新：点击"复盘最新局势"和"试探大法师"后均返回新的后续行动，不再重复已执行的行动。

## v1.2 - 2026-05-24

### Story Bible 与故事生成

- 重构故事分析与适配逻辑，让事件、角色、阵营、能力和 UI 面板更贴合玩家输入的故事设定，减少硬套固定题材模板的问题。
- 强化 Story Bible 生成与校验，检查角色目标、NPC 知识范围、Widget 配置、结局数量和不安全的完整故事访问权限。
- 改进 Demo 与 Mock 生成路径：Demo 可以保留可玩兜底，导入故事在 AI 失败时明确提示未生成，不用模板剧情冒充真实生成。

### AI GM、Rule Engine 与 World State

- 将玩家行动结果传入 AI GM 上下文，使中间叙事框能结合行动、规则结算、World State、触发事件和新增情报生成具体反馈。
- 修复玩家可见文本中的内部 ID 泄露问题，包括事件 ID、目标别名、指标 key、玩家 ID 等。
- 在规则结算前统一规范行动目标，支持 NPC、角色、事件、当前位置和关联地点等目标别名。
- 改进社交、调查、自我梳理、线索、指标、关系和事件结算，让行动产生更具体的结果，而不是只显示“成功/失败”。
- 保持核心约束：所有世界状态变化都必须经过 Rule Engine，AI GM 只负责叙事、建议事件、揭示信息和推荐行动。

### 玩家交互与结算速度

- 推荐行动和自由行动提交后增加即时 pending 反馈，让玩家明确知道点击已经生效。
- 行动结算后根据最新 World State 刷新推荐行动，并过滤掉刚刚使用过的旧行动。
- 修复推荐行动列表不实时更新的问题：点击后立即从 UI 移除旧行动，结算完成后用新推荐替换；提交失败时恢复原列表。
- 优化行动结算链路：World State 更新后并行执行结局判断和 GM 叙事生成，减少串行等待。
- 行动后的 GM 叙事采用并发高 token AI 请求，优先使用第一个合法 JSON 结果，在不降低文本质量的前提下降低等待时间。
- Demo 的兜底等待时间缩短；导入故事仍保持严格策略，AI 失败时明确显示未生成。

### 玩家界面

- 修复 GM 名称被错误显示为“通用汽车”的问题。
- 已知事实支持展开查看，避免“还有 N 条”但无法查看完整情报。
- 改进行动面板、自由行动输入框和 UI Builder 的禁用、处理中、推荐刷新状态。
- 清理证据、已知事实、角色信息和叙事文本中的内部参数，让玩家看到的是故事语言。

### Demo：失落圣杯之夜

- 打通 Demo 闭环：创建房间、选择角色、开始故事、提交推荐/自由行动、GM 叙事、推荐行动刷新、事件触发和结局判断。
- 强化与大法师等 NPC 交谈后的反馈，让重复对话也能产生可用线索和后续行动方向。
- Demo 允许使用可玩兜底叙事；导入的自定义故事不会用兜底模板替代 AI 生成。
